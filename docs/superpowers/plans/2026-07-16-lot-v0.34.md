**Tech Stack:** Rust (tokio, iroh/QUIC, Windows Media Foundation/WGC/NVENC pour la vidéo), TypeScript
(ES modules natifs, pas de framework), Tauri `invoke`/`listen` + `Channel` binaire.

# Jalon M2 — Item 3 : préréglages qualité/fps du partage natif

> **DÉCISION 2026-07-16 (livraison) — FPS-ONLY.** À l'implémentation, on a constaté que
> l'étage BGRA→NV12 (`bgra_to_nv12`) ne fait que rogner/letterboxer 1:1, sans redimensionnement :
> la downscale de résolution corromprait les trames. Sur choix utilisateur, la **downscale de
> résolution (2K/1080p/720p sous le natif) est REPORTÉE** à un lot ultérieur (elle exige un vrai
> étage de mise à l'échelle). **Ce lot ne livre que le plafond de fps** ; la résolution reste
> native. Conséquences : Task 2.1 (U2) simplifiée en `Quality { fps }` (le plafonnement de
> résolution/`clamp_dims` retiré) ; Task 2.2 ne garde que `maxFps` ; Task 2.3 devient un
> **sélecteur de fps** (pas de préréglages de résolution). Les sous-sections ci-dessous décrivant
> `maxW/maxH` et les préréglages 2K/1080p/720p sont **superseded** par cette décision.

**Problème :** `FPS` est une constante (30), la résolution = native, `LEVELS` part de (30,100 %).
Aucun choix utilisateur. Ajouter un plafond (fps + résolution cible, jamais d'upscale) choisi dans le
picker et propagé jusqu'à l'encodeur ; l'adaptatif reste libre de descendre.

```rust
    /// Dimensions d'encodage : plafonne au ratio, JAMAIS d'upscale, force la parité (NV12).
    /// (0,0) en plafond = illimité → résolution native.
    pub(super) fn clamp_dims(nw: u32, nh: u32, mw: u32, mh: u32) -> (u32, u32) {
        let (mut w, mut h) = (nw, nh);
        if mw > 0 && mh > 0 && (nw > mw || nh > mh) {
            let scale = f64::min(mw as f64 / nw as f64, mh as f64 / nh as f64);
            w = (nw as f64 * scale).round() as u32;
            h = (nh as f64 * scale).round() as u32;
        }
        (w & !1, h & !1) // parité
    }

    /// Échelle d'adaptation relative à la cadence cible (niveau 0 = cible/100 %).
    pub(super) fn levels_for(fps: u32) -> [(u32, u32); 4] {
        let f = fps.max(1);
        [
            (f, 100),
            ((f * 2 / 3).max(1), 66),
            ((f * 2 / 5).max(1), 40),
            ((f / 4).max(1), 25),
        ]
    }
```
